# Sign Up Form Validation
## Entri Elevate Module End Assignment (11)
### By Ashfaaq Feroz Muhammad

This project is a sleek, responsive Sign Up form built with HTML, CSS, Bootstrap, and JavaScript. It features client-side validation to ensure data integrity and user experience.

## Project Structure
```
signup-form-validation/
├── index.html       # Main HTML file containing the form
├── css/
│   └── styles.css   # Custom CSS styles (Blue gradient theme)
├── js/
│   └── validation.js # JavaScript for form validation
└── README.md        # Project documentation
```

## Features
- **Responsive Design**: Uses Bootstrap 5 for a layout that works on all devices.
- **Client-Side Validation**: HTML5 and JavaScript validation to check inputs before submission.
- **Password Matching**: Javascript ensures the password and confirm password fields match.
- **Visual Feedback**: Real-time distinct visual cues for valid and invalid states.
- **Aesthetic**: Modern blue gradient background with a clean card interface.

## Technologies Used
- **HTML5**: For structure and semantic elements.
- **CSS3**: For custom styling and gradients.
- **Bootstrap 5**: For responsive grid, components, and utility classes.
- **JavaScript**: For logic and validation handling.

## How to Run
1. Navigate to the project folder.
2. Open `index.html` in any modern web browser (Chrome, Firefox, Safari, etc.).
3. Fill out the form to test the validation features.

## Validation logic
- **Full Name**: Required.
- **Email**: Required, must be a valid email format.
- **Password**: Required, minimum 6 characters.
- **Confirm Password**: Must match the Password field.
- **Terms**: Must be checked.

## Conclusion
This assignment demonstrates the implementation of a robust sign-up form, focusing on both usability and security through proper validation techniques.
